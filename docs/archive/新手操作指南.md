# 新手操作指南（从零开始）

> **已归档（2026-07-11）**：精简版见 [`../立得香港看板-AI助手交接指南.md`](../立得香港看板-AI助手交接指南.md) §4.1。本文部分账号/入口描述已过时。

这份文档假设你**第一次**做这类项目，按顺序做即可。每一步做完再进行下一步。

---

## 你先需要弄清的三件事

1. **这个系统是什么**
  一个网站：销售登录后看自己负责的商户，交易变少会自动预警。  
   数据不是接支付 API，而是你们从上游后台**导出 Excel/CSV**，由管理员上传。
2. **两类账号**
  - **管理员**：上传交易文件、改预警比例（例如下降 30% 报警）  
  - **销售**：只能看自己名下的商户和预警
3. **项目放在哪**
  文件夹路径：`/Users/Eric/projects/sales-data-agent`

---

## 第一步：安装必备软件

### 1. 安装 Node.js（必装）

1. 打开浏览器访问：[https://nodejs.org/](https://nodejs.org/)
2. 下载 **LTS（推荐）** 版本，双击安装，一路「继续」即可。
3. 安装完成后，打开 **终端**（Mac：聚焦搜索输入 `终端` 或 `Terminal`）。

在终端里输入下面两行（每行回车一次），确认有版本号输出：

```bash
node -v
npm -v
```

若显示类似 `v22.x.x` 和 `10.x.x` 就说明成功。

### 2. ~~安装 Xcode 命令行工具~~（已不需要）

本项目已改用 **Node 自带数据库**，一般**不用**再装 Xcode。  
若你之前装失败、提示「无法找到服务器」，**可以跳过这一步**。

---

## 第二步：进入项目并安装依赖

在终端**依次**复制粘贴执行：

```bash
cd /Users/Eric/projects/sales-data-agent
```

```bash
npm install
```

- 第一次会下载很多包，需要 **2～10 分钟**，请耐心等待。  
- 若最后没有红色 `error`，一般就是成功。

---

## 第三步：初始化演示数据（只需做一次）

```bash
npm run db:seed
```

成功时终端会提示创建了账号，例如：

- 管理员：`admin` / `admin123`  
- 销售：`zhangming` / `sales123`（代表「张明」）  
- 销售：`lifang` / `sales123`（代表「李芳」）

---

## 第四步：启动网站

```bash
npm run dev
```

看到类似下面提示就说明启动成功（**不要关这个终端窗口**）：

```
✅ 请在浏览器打开：
   http://localhost:3080
```

1. 打开浏览器（Chrome / Edge / Safari 均可）
2. 地址栏输入：**[http://localhost:3080](http://localhost:3080)** 然后回车

> ⚠️ 请用 **3080** 端口，不要用 5173。只开网页、没开后端时登录会报「请求失败 404」。

---

## 第五步：用管理员体验「上传数据」

1. 在登录页输入：
  - 用户名：`admin`  
  - 密码：`admin123`
2. 点 **登录**
3. 右上角点 **数据管理**
4. 先点 **下载导入模板样例**，打开看看列长什么样（商户名称、交易名称、交易时间、交易金额、销售）
5. 点 **选择文件上传**
  - 可先选项目里的样例文件：  
   `sales-data-agent` → `public` → `samples` → `transactions.csv`  
  - 导入方式选 **全量替换**（第一次推荐）
6. 上传成功后会有绿色提示

回到首页（点左上角或浏览器后退），你会看到 **交易预警** 和 **我的商户** 列表。

---

## 第六步：用销售账号体验「销售只看自己的」

1. 点右上角 **退出**
2. 用销售账号登录：
  - 用户名：`zhangming`  
  - 密码：`sales123`
3. 列表里应主要是「张明」负责的商户（样例数据里销售列写的是「张明」）
4. 点某个商户 **详情**，可看：
  - **日 / 周 / 月** 环比卡片  
  - 交易趋势图  
  - 每一笔交易明细

再退出，用 `lifang` / `sales123` 登录，看到的商户会不一样。

---

## 第七步：改预警规则（例如「月环比下降 30% 才报警」）

1. 用 **admin** 重新登录
2. **数据管理** → 下方 **预警阈值**
3. 在「月环比预警」里填 `30`，点 **保存并重算**
4. 回首页看预警列表是否更新

含义：**本月**交易额比**上月**少 ≥30% 的商户，会出现在预警里（周、日同理）。

---

## 第八步：以后用你们自己的真实数据

1. 登录上游支付公司 **后台**
2. 导出 **全部商户交易明细**（CSV 或 Excel 均可）
3. 确认表里至少有这些列（列名中英文类似即可）：
  - 商户名称  
  - 交易名称  
  - 交易时间  
  - 交易金额  
  - **销售 / 业务员**（必须和你们系统里销售姓名一致，如「张明」）
4. 用 **admin** 登录本系统 → **数据管理** → 上传该文件 → 选 **全量替换**

> **重要**：导出表里的「销售」姓名，要和系统里销售账号的 **显示名** 一致，否则销售登录后看不到商户。  
> 目前样例里「张明」对应账号 `zhangming`，「李芳」对应 `lifang`。

---

## 第九步：给销售开通账号（管理员在网页操作）

1. 用 **admin** 登录 → **数据管理**
2. 找到 **销售账号** 区域
3. 填写并 **添加销售账号**：
  - **登录用户名**：如 `jasonlee`（销售登录用）  
  - **显示名**：必须与智付数据一致，如 `JasonLee`（与文件名 `JasonLee4-6月交易明细.xlsx` 或表内销售列相同）  
  - **初始密码**：至少 6 位，私下发给销售
4. 导入交易数据后，点 **同步商户归属**（把商户挂到对应销售）
5. 销售用分配的用户名密码登录，只能看到自己名下商户

已有演示账号：`sam` / `winnie` / `sales123`，显示名 Sam / Winnie。

---

## 第十步：给所有销售在家/外面用（外网上线）

本地 `http://localhost:3080` **只有本机**能访问，销售在外面打不开。

请按项目内 `**docs/外网部署指南.md`** 操作（Docker + 云服务器 + 域名 HTTPS），部署完成后把地址发给销售，例如：

`https://sales.你的公司.com`

销售用手机浏览器打开即可，**不需要连公司局域网**。

## 常见问题


| 现象                              | 处理                                                                                     |
| ------------------------------- | -------------------------------------------------------------------------------------- |
| `npm install` 报 `gyp` / `Xcode` | 先删旧依赖：`rm -rf node_modules package-lock.json`，再 `npm install`；仍失败把完整报错发给我              |
| 登录报 404                         | 必须用 `npm run dev` 启动，浏览器打开 **[http://localhost:3080](http://localhost:3080)**（不是 5173） |
| 页面打不开                           | 确认 `npm run dev` 的终端还在运行，地址是 `http://localhost:3080`                                   |
| 销售登录后没有商户                       | 检查导出 CSV 里「销售」列是否和账号姓名一致                                                               |
| 没有预警                            | 可能交易波动不够大；用 admin 调低阈值（如改成 10%）测试                                                      |
| 关掉终端网站就关了                       | 正常；下次再 `cd` 进项目执行 `npm run dev`                                                        |
