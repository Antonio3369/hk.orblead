# 发版流程（Mac → 腾讯云 OrcaTerm）

生产地址：`http://43.136.25.181:3080`  
Mac 项目：`/Users/Eric/projects/sales-data-agent`  
服务器：`/opt/sales-data-agent`

**不要用 Trae 改服务器。** 看板与 `tools/alipay/` 完全无关。

---

## 只改前端（界面、文案、样式）

### Mac

```bash
cd /Users/Eric/projects/sales-data-agent
npm run pack:frontend
```

输出：`release/sales-dist-YYYYMMDD-HHMM.tgz`

### 上传

把 tgz 传到服务器，例如 `/home/ubuntu/sales-dist-xxx.tgz`

### OrcaTerm

```bash
cd /opt/sales-data-agent
sudo tar xzf /home/ubuntu/sales-dist-xxx.tgz -C /opt/sales-data-agent
sudo docker compose up -d --build
```

验证：

```bash
curl -I http://127.0.0.1:3080/
```

`Last-Modified` 应为刚 build 的时间。浏览器 **强制刷新** 或无痕窗口。

> **不要用 `docker compose restart`**：前端在镜像 build 时 COPY 进容器，`restart` 不会加载新 `dist`。

---

## 改后端或全量更新

### Mac

```bash
cd /Users/Eric/projects/sales-data-agent
npm run pack:deploy
```

输出：`release/sales-agent-YYYYMMDD-HHMM.tgz`

### OrcaTerm

```bash
cp /opt/sales-data-agent/.env /tmp/.env.backup
sudo tar xzf /home/ubuntu/sales-agent-xxx.tgz -C /opt/sales-data-agent
cp /tmp/.env.backup /opt/sales-data-agent/.env
cd /opt/sales-data-agent
sudo docker compose up -d --build
sudo docker compose ps
```

## 必守规则

| 项目 | 说明 |
|------|------|
| `.env` | 只在服务器 `/opt/sales-data-agent/.env`，解压前备份，勿覆盖 JWT |
| 数据 | 在 Docker 卷 `sales-data-agent_app-data`，正常发版不丢 |
| 勿删 | `docker-compose.yml`、`Dockerfile.prod`、`.env` |
| 勿上传 | `tools/`、`secrets/`、`node_modules/`、`data/` |
| 改 JWT | 全员需重新登录 |

## 常见问题

| 现象 | 处理 |
|------|------|
| 服务器 `npm run pack:*` 报错 | 打包只在 **Mac** 做，不在服务器 |
| 页面仍是旧版 | `up -d --build` + 浏览器强制刷新 |
| 网站打不开 | `sudo docker compose ps`，缺 yml 见交接指南 `docs/立得香港看板-AI助手交接指南.md` §三 |
