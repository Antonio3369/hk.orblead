**不要用 Trae 改服务器。** 看板与 `tools/alipay/` 无关。

## 发版门禁

1. **只有用户明确说「部署」**（或「上线到生产」）才执行生产发版；功能说「可以了」≠ 授权部署。
2. **部署前**：沙箱验证关键路径 + 自检（`npm run typecheck`；有依赖变更时 `npm audit`；大改扫 diff/冲突点）。
3. AI / Cursor 不得在未确认时 SSH 改生产或跑 `npm run deploy`。

## 推荐：一键发版（与 ali 同款）

本机已配置好 `sales-cloud` SSH 时：

```bash
cd /Users/xin/projects/hk.orblead
npm run deploy
# 等同：./deploy/push-and-deploy.sh
```

脚本会：

1. 本地 `npm run build`（`Dockerfile.prod` 需要 `dist` / `dist-server`）
2. `rsync` 到服务器 `/tmp/hk-orblead-deploy`（生产目录常非 ubuntu 属主）
3. `sudo rsync` 覆盖 `/opt/sales-data-agent`（**保留** `.env`，不传 `data/` / `tools/` / `numbers/`）
4. `deploy/server-deploy.sh` → `sudo docker compose up -d --build` + 探活

可选：

```bash
SKIP_BUILD=1 npm run deploy          # 已 build 过
DEPLOY_SERVER=sales-cloud npm run deploy
```

验证：浏览器 **强制刷新** https://hk.orblead.com ；或：

```bash
ssh sales-cloud 'curl -sI http://127.0.0.1:3080/ | head -5'
```

**2026-07-18 起建议抽验**（若本次改动涉及导入 / 境外卡）：

1. admin **追加导入** 一份小机构报表（勿覆盖）→ 数秒内返回；整站仍可浏览。
2. 侧栏 **境外卡交易** → 首卡为「**本月**境外卡交易排名 · Top 10」；页面不无限转圈。
3. 后台上传时进度条应持续可见（小文件亦如此）。

> **不要用 `docker compose restart`**：前端在镜像 build 时 COPY 进容器，`restart` 不会加载新 `dist`。

## 备用：打 tgz + OrcaTerm（无 SSH / 网络受限时）

### 只改前端

```bash
npm run pack:frontend   # → release/sales-dist-*.tgz
```

上传后 OrcaTerm：

```bash
cd /opt/sales-data-agent
sudo tar xzf /home/ubuntu/sales-dist-xxx.tgz -C /opt/sales-data-agent
sudo docker compose up -d --build
```

### 全量 / 后端

```bash
npm run pack:deploy     # → release/sales-agent-*.tgz
```

OrcaTerm：

```bash
cp /opt/sales-data-agent/.env /tmp/.env.backup
sudo tar xzf /home/ubuntu/sales-agent-xxx.tgz -C /opt/sales-data-agent
cp /tmp/.env.backup /opt/sales-data-agent/.env
cd /opt/sales-data-agent
sudo docker compose up -d --build
```

## 必守规则

| 项目 | 说明 |
|------|------|
| `.env` | 只在服务器；rsync / 解压都不得覆盖 `JWT_SECRET` |
| 数据 | Docker 卷 `sales-data-agent_app-data`，正常发版不丢 |
| 勿删 | `docker-compose.yml`、`Dockerfile.prod`、`.env` |
| 勿上传 | `tools/`、`secrets/`、`node_modules/`、`data/`、`numbers/` |
| 改 JWT | 全员需重新登录 |
| 构建位置 | **Mac 上 build**；服务器只 `docker compose --build` 打镜像（不在服务器跑 `npm run build`） |

## 常见问题

| 现象 | 处理 |
|------|------|
| `sales-cloud` 连不上 | 检查 `~/.ssh/config`；临时改用 tgz + OrcaTerm |
| 服务器缺 `dist` | 须走本机 `npm run deploy`，不要只 rsync 源码 |
| 页面仍是旧版 | 确认用了 `--build` + 浏览器强制刷新 |
| 网站打不开 | `ssh sales-cloud 'cd /opt/sales-data-agent && sudo docker compose ps'` |
| 导入很慢 / 整站卡死 | 确认已发版 §导入性能（2026-07-18）；用 **append**；勿升配带宽 CPU 代替修代码 |
| 境外卡页一直 loading | 强制刷新；查 `/api/overseas-cards/overview`；旧前端缺字段会白屏（2026-07-18 已兼容） |
