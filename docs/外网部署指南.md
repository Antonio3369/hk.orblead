# 外网部署指南（销售户外用手机/电脑访问）

`localhost` 只有你自己电脑能打开。要让销售在**任何网络**（4G、家里 WiFi、咖啡厅）访问，需要把系统部署到**有公网地址的服务器**，并建议使用 **HTTPS**。

---

## 一、你需要准备什么

| 项目 | 说明 |
|------|------|
| 云服务器 | 阿里云 / 腾讯云 / 华为云等，1 核 2G 内存即可，系统选 **Ubuntu 22.04** |
| 公网 IP | 买服务器时自带 |
| 域名（推荐） | 如 `sales.你的公司.com`，没有域名也可先用 `http://公网IP:3080` 试用 |
| SSL 证书（推荐） | 有域名可用免费 Let's Encrypt，手机浏览器更安全 |

---

## 二、最快方式：Docker 一键部署

### 1. 登录服务器，安装 Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 重新登录 SSH 一次
```

### 2. 把项目上传到服务器

在本机 Mac 上（项目目录）：

```bash
cd /Users/Eric/projects/sales-data-agent
tar czf sales-agent.tgz --exclude=node_modules --exclude=data .
scp sales-agent.tgz root@你的服务器IP:/opt/
```

在服务器上：

```bash
cd /opt && mkdir -p sales-data-agent && tar xzf sales-agent.tgz -C sales-data-agent
cd sales-data-agent
```

### 3. 配置环境变量

```bash
cp .env.example .env
nano .env
```

将 `JWT_SECRET` 改成**至少 32 位随机字符串**（可运行 `openssl rand -hex 32` 生成）。

### 4. 启动

```bash
docker compose up -d --build
```

### 5. 开放防火墙端口

- 云厂商控制台「安全组」：放行 **3080**（或后面 Nginx 用的 80、443）
- 服务器上如有 ufw：`sudo ufw allow 3080`

### 6. 访问

浏览器打开：`http://你的公网IP:3080`  
管理员 `admin` / `admin123`（**上线后请立刻改密码**）。

## 三、绑定域名 + HTTPS（正式给销售用）

建议用 Nginx 做反向代理，只对外暴露 443。

### 1. 安装 Nginx 与证书工具

```bash
sudo apt update
sudo apt install -y nginx certbot python3-certbot-nginx
```

### 2. 域名解析

在域名服务商添加 **A 记录**：`sales.你的公司.com` → 服务器公网 IP。

### 3. Nginx 配置示例

`/etc/nginx/sites-available/sales-agent`：

```nginx
server {
    listen 80;
    server_name sales.你的公司.com;

    location / {
        proxy_pass http://127.0.0.1:3080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用并重载：

```bash
sudo ln -s /etc/nginx/sites-available/sales-agent /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d sales.你的公司.com
```

### 4. 发给销售的地址

`https://sales.你的公司.com`  
手机浏览器收藏此地址即可，**不需要连公司局域网**。

## 四、销售账号怎么配（管理员在网页操作）

1. 用 **admin** 登录已部署的网站  
2. 进入 **数据管理** → **销售账号**  
3. **添加销售账号**：  
   - **登录用户名**：如 `jasonlee`（销售用来登录）  
   - **显示名**：必须与数据一致，如 `JasonLee`（与智付文件名 `JasonLee4-6月…` 或表内销售列一致）  
   - **初始密码**：发给销售  
4. 上传/更新交易数据后，点 **同步商户归属**  
5. 销售用 `https://你的域名` + 自己的用户名密码登录，只能看到本人商户

## 五、日常运维

| 事项 | 做法 |
|------|------|
| 更新交易数据 | admin → 数据管理 → 上传智付 Excel（全量替换或追加） |
| 新增销售 | 数据管理 → 销售账号 → 添加 → 同步商户归属 |
| 改销售密码 | 数据管理 → 对应销售 → 重置密码 |
| 备份数据库 | 复制服务器上 `data/app.db`（Docker 卷在 `app-data` 卷内） |
| 重启服务 | `docker compose restart` |

## 六、常见问题

| 现象 | 处理 |
|------|------|
| 外网打不开 | 检查安全组是否放行端口；`docker compose ps` 是否在运行 |
| 销售登录后无商户 | 显示名是否与 `sales_name` 一致；点「同步商户归属」 |
| 只想 HTTPS 不想暴露 3080 | 安全组只开 80/443，3080 仅本机 Nginx 访问 |
| 数据丢了 | 恢复备份的 `app.db` |

## 七、不用 Docker 的手动部署（可选）

```bash
cd /opt/sales-data-agent
npm install
npm run build
export JWT_SECRET=你的随机密钥
export PORT=3080
npm run db:seed   # 仅首次
npm start
```

同样用 Nginx 反代到 3080，步骤与上文第三节相同。
